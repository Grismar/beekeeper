The Beekeeper plugin is an embedded web server for Musicbee. It exposes most of the Musicbee API as an http interface, or a web API. Using the web API is easy for programmers in non-.Net languages. You can even directly access the API itself from your browser!

To make things more convenient, the Beekeeper plugin can also serve up files you put in the Musicbee persistent storage folder. If the folder doesn't exist, just create it and put any files you need served there. If you don't know where the folder should be, try visiting /file/fakename and the error message will tell you where to look.

The current version of Beekeeper supports MusicBee API revision 43. To see if everything is working as it should be, or to find out how you can use the API yourself, put the "test" folder in the persistent storage folder and then visit it at /file/test/index.html. Please check the contents of the folder for more information and make sure you have the version matching the running Beekeeper version.

How to use

One way to use the Beekeeper plugin is to write your own web interface for Musicbee, or perhaps write an application in another language that uses the web API. Another way is to look for someone who did just that and is making the result available for you to use. For example, you could use the Beekeeper Android app.

Web API

The Web API consists of two important parts: MusicBee event notifications and MusicBee methods. You can retrieve recent event notifications at /events. It returns a serialized JSON object containing all of the events notifications since the last request for /events. If no notifications are available, Beekeeper will wait for a maximum of 5 seconds and if an event occurs in those 5 seconds, it will immediately respond. By chaining requests for /events together, you can make sure you always have the most recent events. For more details on notifications, see the "test" folder. The methods of the MusicBee API can be called by posting to the appropriate URL, with parameters in a stringified JSON object. For example, you can visit /Player_PlayPause to play or pause the current track and it will return "null", since there is no function result for the method. HybridDSP.Net.dll

Beekeeper uses the freely available HybridDSP.Net component from Hybrid Geotools. More information about its use in this article on CodeProject. The source of HybridDSP is included with Beekeeper.

You may come across alarming messages online that claim HybridDSP is something evil, hell-bent on destroying your PC. If you're worried, feel free to read the source files. You can even compile the source yourself, using Microsoft's Visual Studio Community edition and use the resulting .dll file instead of the one packaged with Beekeeper. The alarming messages online are probably due to someone using HybridDSP.Net as a part of some malware, but the component itself is fine. Note: Hybrid Geotools have nothing to do with Beekeeper or Musicbee, they were just friendly enough to share their software component.
